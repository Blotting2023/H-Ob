## 一、如何在 Obsidian 中打开

### 1.1 首次设置

```bash
# 数据库路径
/root/行业数据库/创面敷料行业/

# 推荐以 Obsidian Vault 形式打开
# 直接将 "创面敷料行业" 目录拖入 Obsidian → "Open folder as vault"
```

### 1.2 必备插件

| 插件名 | 用途 | 安装位置 |
|--------|------|----------|
| **Dataview** | 表格查询、统计、索引 | 社区插件 → 搜索 "Dataview" |
| **Graph View** | 可视化知识图谱（双向链接网络图） | 内置功能 |
| **Tag Wrangler** | 标签批量管理 | 社区插件 |
| **Calendar** | 按日期浏览笔记 | 社区插件 |
| **Periodic Notes** | 日报/周报/月报模板 | 社区插件（可选） |

### 1.3 建议视图

- **默认视图**: 打开 `00-系统总览.md` 作为仪表盘页面
- **知识图谱**: Ctrl/Cmd + G 打开 Graph View，过滤标签查看特定模块连接
- **文件浏览器**: 左侧展开目录树，按 `01-品牌` → `02-产品` → ... 层级浏览

## 三、如何使用 Dataview 查询

### 3.1 常用查询示例

**查询所有品牌**：
````markdown
```dataview
TABLE 母公司, 核心卖点, 估计规模
FROM "01-品牌/Brands"
SORT file.name ASC
```
````

**查询某个产品类型下的所有品牌**：
````markdown
```dataview
TABLE 品牌, 主要产品, 核心卖点
FROM "01-品牌/Brands"
WHERE contains(主要产品, "泡沫敷料")
SORT file.name ASC
```
````

**查询带特定标签的笔记**：
````markdown
```dataview
LIST
FROM #抗菌
SORT file.ctime DESC
```
````

**查询最近更新的文件**：
````markdown
```dataview
TABLE file.mtime AS "最后更新"
FROM ""
SORT file.mtime DESC
LIMIT 20
```
````

**查询所有带 [待核实] 标签的文件**：
````markdown
```dataview
LIST
FROM ""
WHERE contains(file.outlinks, "[待核实]") OR contains(file.content, "[待核实]")
SORT file.name ASC
```
````

### 3.2 高级查询：按完成度统计

````markdown
```dataview
TABLE length(rows) AS "文件数"
FROM "03-临床痛点与需求"
GROUP BY "临床痛点" AS "模块"
```
````

> 💡 **Dataview 最佳实践**: 所有 YAML frontmatter 字段名用 `::` 分隔的关系，在 dataview 中可自动识别为链接字段。

## 四、双向链接的使用规范

### 4.1 链接规则

| 场景 | 写法 | 示例 |
|------|------|------|
| 引用品牌 | `[[品牌名]]` | `[[ConvaTec]]` 是主要竞品 |
| 引用产品 | `[[产品类型名\|显示名]]` | `[[01-泡沫敷料-Foam-Dressings\|泡沫敷料]]` |
| 引用痛点 | `[[痛点描述]]` | 见 `[[01-换药疼痛]]` |
| 引用外部URL | `[显示名](URL)` | [PubMed PMID 12345678](https://pubmed.ncbi.nlm.nih.gov/12345678) |
| 引用法规 | `[[法规文件]]` | 参考 `[[FDA注册路径]]` |

### 4.2 链接密度要求

- 每个品牌/产品文件**至少链回**到对应数据库索引页
- 每个商业机会**至少引用**其依赖的技术路线和痛点
- 所有外部来源**必须**包含完整超链接
- 善用 `[[文件]]^锚点` 进行段落级链接（如 `[[10-商业机会]]^机会1`）

### 4.3 知识图谱维护

- 新文件创建后，检查 Graph View 是否形成了足够的连接
- 如果孤立节点（无入链/出链）超过3个，说明需要补充交叉引用
- 推荐最低出链数：每个文件 ≥ 2条出链

## 五、数据标注规范

### 5.1 标签体系

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| `[待核实]` | 数据未经人工验证 | 所有 AI 自动填充的数据 |
| `[待填充]` | 字段暂时为空 | 临时占位 |
| `[2026Q2]` | 数据来源时间戳 | 标注数据时效性 |
| `[已核实:YYYYMMDD]` | 已人工验证 | 确认无误后替换 `[待核实]` |
| `[已过时]` | 数据超过6个月需更新 | 标注需定期刷新数据 |
| `[关键竞品]` | 核心监控对象 | 品牌文件中使用 |
| `[高优先级]` | 需尽快处理 | 紧急任务标记 |

### 5.2 标注规范示例

```markdown
# 某品牌信息

## 市场规模
- 全球创面敷料市场约 **XX 亿美元**（2025）[待核实] [2025Q4]
- 中国市场约 **YY 亿元**（2025）[已核实:20260315]

## 新产品
- 某公司发布新产品 Z（2026-05）[已过时—需确认是否仍在售]
```

### 5.3 内部链接标注

```markdown
- 与 [[竞品品牌名]] 形成直接竞争 ([待核实:竞争关系])
- 可解决 [[01-换药疼痛]] 和 [[05-频繁更换]] 两大痛点
- 使用 [[08-专利地图/两性离子涂层专利专题.md]] 中的核心技术
```

## 六、与 Codex/Agent 的协作工作流

### 6.1 日常协作流程

```
用户指令 → Agent 理解需求 → 搜索数据库 → 生成结构化回答 → 用户审查修正 → 回写到数据库
```

### 6.2 可交给 Agent 的任务

| 任务类型 | 指令示例 |
|----------|----------|
| **批量填充数据** | "基于 PubMed 检索结果，为 [[03-临床痛点与需求/]] 补充分子机制描述，标注 `[待核实]` 标签" |
| **更新趋势数据** | "检查 [[09-趋势与动向/趋势数据库.md]] 中的第3条趋势（两性离子材料），更新2026年Q2最新数据" |
| **竞品动态扫描** | "搜索 2026年5月 Smith+Nephew 的新产品发布和财报数据，更新 [[01-品牌/Brands/Smith-Nephew-竞品深度拆解.md]]" |
| **生成周报/月报** | "按 [[00-情报系统运行手册.md]] 中的周报prompt模板，生成本周行业周报" |
| **数据质量检查** | "扫描全库，列出所有 `[待核实]` 标记的文件清单" |
| **交叉分析** | "找出 [[03-临床痛点与需求/]] 中与 [[08-专利地图/两性离子涂层专利专题.md]] 技术相关的痛点" |

### 6.3 输出格式规范

Agent 输出的数据文件应：
1. 严格遵循对应文件的 frontmatter 和字段模板
2. 所有未经验证数据标注 `[待核实]` + 时间戳
3. 提供原始数据来源链接
4. 使用 Obsidian 内部链接 `[[文件名]]` 建立关联

### 6.4 人工审查窗口

| 任务类型 | 建议审查频率 |
|----------|-------------|
| 自动填充的品牌/产品数据 | 每月一次 |
| 专利监控结果 | 每季一次 |
| 周报/月报 | 每次生成后快速过目 |
| 商业机会评估 | 每季深度审查一次 |
| 系统总览指标 | 每周花2分钟确认 |

## 附录：快速故障排除

| 问题 | 解决方法 |
|------|----------|
| Dataview 查询不返回结果 | 检查文件是否有正确的 YAML frontmatter（`---` 包裹） |
| 双向链接显示为未解析 `[[文件]]` | 检查文件名是否完全匹配（含路径） |
| 标签无法检索 | 标签只能使用 `#tagname` 格式，不含空格 |
| 文件编码错误 | 确保所有文件为 UTF-8 编码 |
| 图片无法显示 | Obsidian 只支持本地图片或公开URL，不支持本地绝对路径图片 |
