# 创面敷料行业情报系统 — 运行手册

> **版本**: v1.0 | **最后更新**: 2026-06-06
> **目标**: 让数据库从"静态资料库"升级为"持续运转的情报系统"
> **作业指导**: 本手册定义日/周/月/季四级运转节奏、自动生成模板、知识地图更新触发条件、以及从数据到决策的方法论

---

## 一、日 / 周 / 月 / 季 四级运转节奏

### 每日（约10分钟）

| 时间 | 任务 | 操作要点 | 输出 |
|------|------|----------|------|
| ☀️ 09:00 | X/Twitter 学术热帖快速扫描 | 搜索 `wound dressing + recent`, 关注 @WoundCareToday @JWCeditor 等账号的当日热帖 | 标记1-2条值得关注的帖子，截取链接 |
| ☀️ 09:10 | FDA 510(k) 新批产品检查 | 访问 [FDA 510(k) 数据库](https://www.accessdata.fda.gov/scripts/cdrh/cfdocs/cfpmn/pmn.cfm)，Product Code= `FRO`/`KGN`/`MGP`/`MGR`，Date=最近1天 | 如有新批产品，记录到 [[06-法规注册/法规数据库.md]] |

> 💡 **每日扫描仅10分钟，严格限时。** 超出时间的信息纳入周度批量处理。

### 每周（约30分钟）

| 任务 | 操作指引 | 耗时 |
|------|----------|------|
| 📄 **PubMed 新文献速览** | 使用 [[04-关键意见领袖/PubMed检索策略.md]] 中保存的检索式，筛选最近7天发表文献 | 10min |
| 🏢 **竞品社媒动态** | 关注 [[01-品牌/Brands/]] 中核心竞品（Molnlycke、Smith+Nephew、ConvaTec、3M等）的LinkedIn/Twitter动态 | 10min |
| 📰 **行业新闻扫描** | 浏览 [[99-原始数据/信息源清单.md]] 中的行业媒体（MedTech Dive、WoundSource、Medical Plastics News等） | 10min |

**周度输出**: 使用下方 **周报模板** 生成一份 Markdown 周报，存入 `99-原始数据/周报/` 目录，文件命名 `WYY-周报-YYYY-MM-DD.md`。

### 每月（约60分钟）

| 任务 | 操作指引 | 耗时 |
|------|----------|------|
| 🔬 **专利新申请分析** | 使用 [[08-专利地图/专利监控策略.md]] 中的检索式，在CNIPA/USPTO/EPO检索最近30天新公开专利 | 15min |
| 📜 **法规 / 集采动态汇总** | 更新 [[06-法规注册/法规动态监控.md]]；检查NMPA新批/FDA新Guidance/集采公告 | 15min |
| 🏷️ **新增品牌 / 产品入库** | 按 [[00-数据库使用指南.md]] 的规范，将新发现的品牌/产品录入对应模块 | 15min |
| 🤖 **月度情报简报** | 让 Codex/Agent 自动生成月度简报（调用下方 Prompt 模板） | 15min |

**月度输出**: 一份 Markdown 简报存入 `99-原始数据/月报/` 目录，文件命名 `MYY-月报-YYYY-MM.md`。

### 每季（约2小时）

| 任务 | 操作指引 | 耗时 |
|------|----------|------|
| 📊 **趋势验证与更新** | 对照 [[09-趋势与动向/趋势数据库.md]] 中的10大趋势，逐一验证假设、更新数据 | 30min |
| 🗺️ **知识地图更新** | 检查 [[00-系统总览.md]] 中模块完成度，确定下一季度数据填充优先级 | 20min |
| 💰 **商业机会重新评估** | 对 [[10-商业机会/商业机会.md]] 中的10个机会，更新优先级评分和市场假设 | 30min |
| 📑 **季报输出** | 整合本季度工作，输出行业季报（含关键数据更新、趋势变化、竞品动态、商业机会评估） | 40min |

**季度输出**: 一份专业水准的行业季报（.md 格式），存入 `99-原始数据/季报/`，文件命名 `QYY-季报-YYYY-Qn.md`。

## 二、自动生成周报模板

> 以下为 **Codex/Agent Prompt 模板**，每周一上午使用，粘贴给 Agent 即可自动生成当周行业周报。

### Prompt 模板（复制给 Codex）

```
# 任务：生成创面敷料行业周报

## 上下文
你是创面敷料行业情报系统，需要根据以下数据源生成当周周报。

## 数据源
- PubMed 检索式: (wound dressing OR wound care) AND (clinical trial OR RCT OR meta-analysis) AND (last 7 days[pdat])
- FDA 510(k) 新批: Product Code in (FRO, KGN, MGP, MGR), 最近7天
- 信息源清单: [[99-原始数据/信息源清单.md]]
- 趋势数据库: [[09-趋势与动向/趋势数据库.md]]

## 周报模板结构

## 📍 本周行业动态（3-5条）
- 列出当周最重要的行业新闻，每条约100字，附来源链接

## 🆕 新产品 / 新批准（1-3条）
- 列出最近获批的新产品，包含：产品名、公司、分类、获批市场、核心特点

## 📄 重要论文（2-3篇）
- PMID: 标题 | 期刊 | 核心发现 | 临床意义

## 🏢 竞品动向（2-3条）
- 主要竞品的融资、收购、新品发布、高管变动等信息

## 🔮 值得关注的新趋势
- 从本周信息中提取1-2个值得关注的早期信号

---

请基于上述信息源，以中文生成本周（{{CURRENT_DATE}}）创面敷料行业周报。仅使用真实可查的数据，如无数据则如实说明。
```

## 三、自动更新知识地图的触发条件

以下事件发生时，应立即（24小时内）更新 [[00-系统总览.md]] 中的知识地图和相关模块：

### 3.1 新技术的出现
- 出现**新的接枝方法**（如新型等离子体接枝、UV诱导接枝工艺）→ 更新 [[08-专利地图/按技术路线/]]，新增技术路线
- 出现**新的涂层材料**（如新型两性离子单体、仿生涂层）→ 更新 [[08-专利地图/]] 和 [[02-产品/产品数据库.md]]
- 出现**新的敷料基材**（如新型发泡工艺、静电纺丝技术突破）→ 更新 [[07-供应链/上游原料/上游原料.md]]
- 出现**智能敷料传感器集成方案** → 更新 [[09-趋势与动向/趋势数据库.md]] 第4条

### 3.2 新竞品的进入
- 知名医疗器械公司进入创面敷料市场 → 更新 [[01-品牌/品牌数据库.md]] 和 [[01-品牌/Brands/]] 新增品牌
- 重大融资的创面敷料初创公司 → 更新 [[01-品牌/品牌数据库.md]] + [[10-商业机会/商业机会.md]] 相关机会
- 国内企业获得重要注册证 → 更新 [[06-法规注册/法规数据库.md]] + [[01-品牌/]]

### 3.3 重大法规变化
- MDR 过渡期延长/缩短 → 更新 [[06-法规注册/法规数据库.md]] 和 [[06-法规注册/CE-MDR/]]
- 中国敷料集采扩围 → 更新 [[06-法规注册/法规数据库.md]] + [[09-趋势与动向/趋势数据库.md]] 第6条
- FDA 新的创面敷料分类指南 → 更新 [[06-法规注册/FDA-510k/FDA注册路径.md]]
- 医保支付政策变化（如CMS变更报销代码）→ 更新 [[09-趋势与动向/趋势数据库.md]] 第8条

### 3.4 新的临床指南发布
- WHO/国际创面愈合组织（WUWHS）发布新指南 → 更新 [[04-关键意见领袖/]]
- 中国卫健委/中华医学会创面管理指南更新 → 更新 [[03-临床痛点与需求/]]
- 重大RCT发表改变临床实践 → 更新 [[09-趋势与动向/]] + [[10-商业机会/]]

### 3.5 触发后的标准动作
1. 在 [[00-系统总览.md]] 中记录触发事件和更新状态
2. 通知团队相关成员
3. 在当周/月报中说明更新内容
4. 标记为 `[已更新:YYYY-MM-DD]`

## 四、从数据库到决策支持

### 4.1 如何使用数据库做产品立项决策

```
决策框架：从「技术→市场→竞争→可执行」四维评估

1. 从 [[09-趋势与动向/趋势数据库.md]] 中找到已验证的长期趋势（如GLP-1影响、基层湿性替代）
2. 从 [[08-专利地图/专利数据库.md]] 确认技术壁垒和自由实施空间
3. 从 [[06-法规注册/法规数据库.md]] 评估注册路径和成本
4. 从 [[10-商业机会/商业机会.md]] 找到匹配趋势的未满足需求
5. 从 [[03-临床痛点与需求/]] 验证真实临床需求强度
6. 从 [[07-供应链/]] 评估原料可及性和成本结构
→ 输出：立项可行性报告
```

### 4.2 如何使用竞品分析做差异化定位

```
路径：从「产品→品牌定位→渠道→未满足需求」四层拆解

1. 从 [[01-品牌/Brands/]] 中选择3-5个核心竞品，阅读深度拆解文件
2. 从 [[02-产品/产品数据库.md]] 对比各竞品的产品矩阵和定价
3. 从 [[03-临床痛点与需求/]] 中找到竞品尚未解决的高频痛点
4. 从 [[05-关键词/]] 中分析竞品的搜索词覆盖和社媒声量
5. 从 [[04-关键意见领袖/]] 中了解竞品与KOL的合作情况
→ 输出：差异化定位矩阵（我们的产品 vs 竞品 vs 临床需求）
```

### 4.3 如何使用痛点数据库做产品迭代

```
迭代路径：从「用户痛点→竞品方案→改进点→验证」闭环

1. 从 [[03-临床痛点与需求/]] 中找到当前产品类别的高频痛点（如换药疼痛、渗液泄漏）
2. 搜索 [[01-品牌/Brands/]] 中各竞品如何解决该痛点
3. 搜索 [[08-专利地图/]] 中是否有更优的技术方案
4. 从 [[05-关键词/电商关键词/电商关键词.md]] 中验证痛点是否在消费者评论中反复出现
5. 从 [[04-关键意见领袖/临床专家/]] 中确认临床专家的真实痛点反馈
→ 输出：产品迭代需求列表（优先级排序）
```

### 4.4 如何使用趋势数据库做战略规划

```
战略路径：从「趋势→影响→机会→行动」四步法

1. 从 [[09-趋势与动向/趋势数据库.md]] 中识别3-5个影响行业的关键趋势
2. 评估每个趋势对自身业务的 SWOT 影响
3. 从 [[10-商业机会/商业机会.md]] 中找到匹配趋势的机会点
4. 从 [[07-供应链/]] 评估能否快速建立供应优势
5. 从 [[04-关键意见领袖/]] 中寻找战略合作伙伴或学术代言人
→ 输出：12-24个月战略路线图
```
